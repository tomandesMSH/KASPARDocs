Broken FilePicker

explorer files affecting other apps

Windows Version: Win11 25H2

File Explorers built in function "filepicker" (aka MSVCP140.dll) was updated, making older software unusable.

Some programs might struggle, render the file as not existing or the program crashes straight upon opening the dialog. This bug is present in a lot of programs (ex. Wondershare Filmora - Editing software)

DISM or sfc doesnt fix this since technically it is the programs fault

Solution:
Copy msvcp140.dll from C:\Windows\System32\ into the programs folder and replace it if possible.
If the program is closed entirely or "one-file", report it to the developers, or good luck reverse engineering it.

If the file picker is broken globally, you can download a working version here, but I recommend using a copy from your own system.
